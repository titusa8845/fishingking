# 🎣 義方金魚王：UX 夜市及時排行系統實作規劃書 (Project Plan)

## 1. 專案願景 (Project Vision)
在「UX 夜市」活動中，透過「撈金魚」的趣味互動，建立一個具備「賽博夜市」風格的及時排行系統。利用 Google Sheets 作為輕量後端，結合 GitHub Pages 前端展示，營造熱鬧、競爭且充滿幽默感的現場氛圍。

## 2. 核心功能與 UX 設計 (Core Features & UX)
### A. 前端輸入端 (/input)
*   **介面設計**：大按鈕設計（+1, +5, -1），適合手機單手操作。提供 +5, +10 快捷鍵。
*   **防呆機制**：提交後進入 Loading 狀態，防止重複點擊。
*   **密碼保護**：簡單的 Admin Key 用於存取重置與刪除功能。

### B. 排行榜展示端 (/leaderboard)
*   **2-1-3 賽跑領獎台**：前三名以直式長條圖（Vertical Bar Chart）呈現。
    *   **第一名**：居中、最高、霓虹金 (#fbbf24)、上方帶有閃爍皇冠。
    *   **第二名**：居左、次高、霓虹銀 (#94a3b8)。
    *   **第三名**：居右、最低、霓虹銅 (#b45309)。
*   **動態效果**：使用 Framer Motion 處理排名變動時的平滑位移。
*   **列表分流**：第四名以後以清單形式在下方展示。

### C. 榮譽插播與嗆聲系統 (Broadcast & Heckling System)
*   **全場廣播 (Overlay)**：當玩家達到 50+ 或 100+ 隻魚，或超越第一名時，觸發全螢幕遮罩。
*   **稱號連動**：
    *   10-30 隻：🎣 垂釣學徒
    *   31-50 隻：🏅 夜市常客
    *   51-99 隻：👑 新王登基 (觸發霓虹金特效)
    *   100+ 隻：👾 水缸終結者 (觸發賽博粉紅特效)
*   **字幕嗆聲**：底部黑色半透明字幕條，搭配青色文字與「打字機效果」。
    *   *範例：「老闆說他準備要把攤位頂讓給你了...」*

## 3. 技術規格 (Technical Specs)
### A. 技術架構
*   **前端 (Frontend)**: React / Vite + Tailwind CSS + Framer Motion。
*   **後端 (Backend)**: Google Sheets。
*   **中繼層 (API)**: Google Apps Script (GAS) 部署為 Web App。
*   **部署 (Hosting)**: GitHub Pages。

### B. 管理後台 (Admin Panel)
*   顯示所有原始數據表格。
*   具備「單筆刪除」功能（處理惡作劇數據）。
*   具備「一鍵重置」按鈕（清空試算表並保留標題列）。
